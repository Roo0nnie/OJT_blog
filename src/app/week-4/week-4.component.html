Introduction

1

Week 4 was another exhausting yet productive week, as it involved the development of key system modules. One of my primary tasks was working on the Admin Panel and User Management, where I implemented Create, Read, Update, Delete, and Search (CRUDS) operations. These are essential for ensuring a dynamic and manageable system.

2

I started by implementing the Create operation, allowing admins to register new moderator accounts. The Read operation was designed to fetch and display user details. The Update function enabled dynamic changes in user status, while the Disable function prevented inactive users from interacting with the system.

3

Beyond CRUD operations, I focused on improving the User Management Interface using Express.js for the backend and an interactive, responsive frontend. To enhance efficiency, I added search and filter functionalities, allowing admins and moderators to quickly locate users based on name, email, account status, or activity level. Additional pagination, status filtering, and list customization improved overall navigation.

4

With User Management in place, I shifted my focus to the Admin and Moderator Login System, a crucial security feature. I implemented user sessions stored in local storage, ensuring that users remain logged in after a page reload and are only logged out when they manually click "Logout." Fine-tuning this functionality took two days.
