Introduction

Week 4 was another exhausting yet productive week, as it involved the development of key system modules. One of my primary tasks was working on the Admin Panel and User Management. This included implementing Create, Read, Update, Delete, and Search (CRUDS) operations, which are fundamental in any system for dynamic and manageable data.

I began by developing the Create operation, allowing admins to create new moderator accounts by entering their information. The Read operation was designed to fetch data from a specific table, enabling both admins and moderators to view user details. The Update operation allowed dynamic changes in user status based on activity, while the Disable function restricted inactive users from interacting with the system.

In addition to backend development using Express.js, I also took charge of designing an interactive and responsive User Management interface for the frontend. However, CRUD operations alone were not enough for efficient user management. To enhance functionality, I integrated search and filter features. The search function allows admins and moderators to quickly find users by name, email, account status, or active status. Similarly, the filter function includes pagination, status filtering, and list customization, making navigation more efficient. Since the CRUD operations for User Management were successfully implemented, I realized that applying the same logic to other modules—such as Task Management and User Communication—would be much easier.

With this progress, I moved on to the Admin and Moderator Login System, a crucial feature ensuring data security and preventing unauthorized access. I developed this functionality by implementing user sessions, which are stored in local storage. This ensures that the admin or moderator remains logged in even after a page reload and only gets logged out upon clicking the "Logout" button, which destroys the session. Developing this feature took me two days to fine-tune according to my requirements. Lastly, I worked on exporting data to PDF by integrating the jspdf library via npm, allowing admins and moderators to generate downloadable reports.
