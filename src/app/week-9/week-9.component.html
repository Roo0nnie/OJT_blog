Access Limitation

Access to core features such as posting and accepting tasks is restricted unless the user's ID and required documents have been verified by the admin. If a user has not uploaded their identification, a "missing information" message will be displayed to notify them. This ensures only authorized users can fully interact with the system and helps maintain security and data integrity.

Login Design

The login system was redesigned to better suit the mobile app environment by removing the email confirmation link, as the system now sends a 6-digit verification code upon login. Upon the user's first successful login, their account status automatically changes from "pending" to "active," simplifying the onboarding process. This update ensures a smoother client registration flow while maintaining a secure and efficient user verification process.

Discord Meeting

The team agreed to revise the design approach for tasker accounts to improve usability and maintain system consistency. The session provided valuable insights and alignment among team members for smoother collaboration.
