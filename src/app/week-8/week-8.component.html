Week 8: SECURITY FEATURES
MARCH 24 - MARCH 28

Security Issue

A security issue was encountered during admin and moderator login due to missing session tokens in HTTP header requests. The Angular configuration lacked a header setter, which initially caused login failures. After troubleshooting, the issue was resolved by implementing a consistent session-based authentication approach across both Flutter and Angular, ensuring secure and seamless login functionality across platforms.

Image Upload Issue (Client/Tasker)

The system faced an error while uploading and fetching user-submitted PDF documents, preventing proper verification status updates. This issue was due to the system's condition not recognizing the PDF file type, which was later addressed by refining the upload validation logic. As a result, users are now properly notified if their profile is incomplete, ensuring smoother profile verification for both clients and taskers.

Account Validation

The account creation process now includes stricter document validation. Taskers are required to upload a TESDA certificate to verify their qualifications for specific tasks, which must be approved by a moderator. Clients, on the other hand, must submit a valid ID for identification purposes, also subject to moderator approval.
